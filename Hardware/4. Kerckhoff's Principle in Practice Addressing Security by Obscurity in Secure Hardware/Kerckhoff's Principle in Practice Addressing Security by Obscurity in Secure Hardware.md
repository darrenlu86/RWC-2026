## Abstract

本演講探討安全硬體（smartcard、TPM、HSM 等）領域中「安全靠隱匿」（security by obscurity）的問題。安全認證體系（Common Criteria、FIPS 140、EMVCo）在攻擊潛力評分中獎勵保密性，形成廠商隱匿設計的動機。研究者提出兩種跨越「隱匿鴻溝」的技術：利用旁通道攻擊反向工程實作細節（將實作空間變成金鑰空間），以及透過向裝置輸入惡意曲線參數（3Q 方法）來識別防禦措施。在多款智慧卡上實驗驗證，證明隱匿並非有效的安全屏障。

## Summary

### 安全硬體的隱匿文化

**安全硬體**涵蓋智慧卡、TPM、HSM、U2F token、安全元件等，廣泛用於信用卡、eID、SIM 卡、手機等。這些裝置底層共享相同的硬體與韌體元件。

**JavaCard 平台**是許多安全硬體的底層架構——在智慧卡上執行 Java 子集，提供硬體加速密碼運算的 API，applet 可實現 SIM、EMV 信用卡等功能，且跨廠商通用。

安全硬體領域的特徵是**極度保密**：大量 NDA、難以取得最新硬體樣本、缺乏公開文件。

### 安全驗證的冰山模型

如何驗證安全硬體的安全性？講者以冰山比喻三個層級：

| 層級 | 安全驗證方式 | 現實狀況 |
|------|-----------|---------|
| 密碼學理論（原語/協議設計） | 開放設計、公開審查競賽（AES、PQC）、形式化證明 | **很少失敗** |
| 開源密碼學實作 | 公開審查、稽核、形式化驗證 | **有漏洞**（CVE 的來源） |
| 安全硬體 | 簽 NDA 自行分析、信任廠商、安全認證 | **Here be dragons**（龍的領域） |

### 安全認證獎勵保密

安全認證（**Common Criteria**、**FIPS 140**、**EMVCo**）在政府、醫療、eID 等領域被強制要求使用。認證本質上是獨立實驗室驗證產品是否符合其安全目標（security target）。

然而，認證體系中的**攻擊潛力評分**（Attack Potential）機制存在根本性問題：

- 攻擊若需要**非公開資訊** → 分數更高
- 攻擊若需要**取得產品樣本** → 分數更高
- 分數足夠高的攻擊 → **不需要緩解**即可通過認證

這直接激勵廠商避免公開任何資訊、避免提供樣本——**認證體系系統性地獎勵保密**。

### 已知漏洞歷史

儘管有認證保護，安全硬體仍反覆出現高影響力漏洞：

| 漏洞 | 年份 | 類型 |
|------|------|------|
| **ROCA** | 2017 | 可分解的 RSA 金鑰 |
| **Minerva** | 2019 | ECDSA 簽章的 timing 洩漏導致金鑰恢復 |
| **TPM-Fail** | 2019 | ECDSA 簽章的 timing 洩漏導致金鑰恢復 |
| **Side journey to Titan** | 2021 | ECDSA 簽章的 EM 洩漏導致金鑰恢復 |
| **TPMScan** | 2024 | ECDSA 簽章的 timing 洩漏導致金鑰恢復 |
| **EUCLEAK** | 2024 | ECDSA 簽章的 EM 洩漏導致金鑰恢復 |

共通點：**全部影響已認證裝置**、**實務上極難修補**（部署後通常無更新路徑）、大多數需要恢復關於目標實作的非公開資訊。

### 安全靠隱匿的鴻溝

旁通道攻擊論文通常假設白盒模型——假設已知純量乘法器類型、加法公式、橢圓曲線模型等。但真實安全硬體對攻擊者而言是**帶旁通道的黑盒**。

這產生了一個**安全靠隱匿的鴻溝**（security by obscurity gap）：攻擊假設白盒，目標卻是黑盒。

兩個關鍵問題：
1. **鴻溝有多大？**——有多少種可能的實作方式？
2. **跨越有多難？**——能否反向工程這些選擇？

### 實作空間的巨大多樣性

以橢圓曲線密碼學為例，實作選擇的組合空間極其龐大：

- **曲線模型**：Short Weierstrass、Montgomery、Twisted Edwards 等，以及它們之間的轉換
- **座標系統**：仿射、射影、Jacobian 等多種選擇
- **加法公式**：每種曲線模型有多種不同的加法公式
- **純量乘法器**：整個研究領域——left-to-right、right-to-left、window method、comb 等
- **有限體運算**：Montgomery、Barrett 等
- **旁通道防禦措施**：多種不同的隨機化方式

組合起來，理論上有**數十萬到數億種**可能的實作。

研究者分析了 18 個開源密碼學函式庫，確認這種多樣性在實務中確實存在：混合曲線模型、各種純量乘法器、多種座標系統、以及大量文獻中未見的自訂加法公式。安全硬體預期有類似的多樣性。

### 技術一：利用旁通道反向工程實作

核心思路：**將攻擊反轉**。傳統旁通道攻擊假設已知實作，目標是金鑰。反向工程則假設已知金鑰（或可控制金鑰），目標是實作。**實作空間變成金鑰空間**。

利用**特殊點攻擊**（RPA、ZVP、EPA）——這些攻擊偵測純量乘法計算中是否出現特定的零值座標點。不同的純量乘法器在計算過程中會經過不同的中間點序列。

**方法**：
1. **離線模擬**：對所有已知的純量乘法器，模擬其在特殊點攻擊下的行為，建立決策表
2. **建立決策樹**：遞迴選擇最佳分裂點，建構判別不同乘法器的決策樹
3. **線上測試**：對目標裝置逐步走訪決策樹，在每個節點發動攻擊，根據結果判斷分支，最終識別出純量乘法器類型

### 技術二：3Q 方法——向實作說謊

實際安全硬體會使用**純量隨機化防禦措施**，阻止上述旁通道反向工程。但這些防禦措施只在**合法定義的曲線**上正確運作。

**JavaCard 平台允許指定任意 Short Weierstrass 曲線參數**——攻擊者可以輸入惡意曲線。

**3Q 方法**：
1. 選擇一條階為 3Q（Q 為大質數）的曲線
2. 向目標聲稱曲線階為 Q（質數階）
3. 輸入一個階為 3Q 的點
4. 觀察輸出結果的分布

不同的防禦措施在這種欺騙性輸入下會產生**不同的輸出分布**——三種可能的結果，每種防禦措施對應不同的機率分布。多次執行後，統計輸出分布即可識別防禦措施類型。

**此方法完全不需要旁通道量測**——純粹透過輸入/輸出行為即可完成。

### 實驗結果

在多款智慧卡上進行實驗（ECDH 和 ECDSA 簽章/金鑰生成），成功識別出：
- 各卡片使用的**防禦措施類型**
- 隨機遮罩值的**位元長度**
- 透過其他方法，甚至可恢復**遮罩的實際值**

### 結論與倡議

1. **安全靠隱匿不可靠**：在橢圓曲線密碼學領域已成功證明
2. **隱匿性難以量化**：旁通道攻擊可被量化評估，但反向工程的難度難以衡量
3. **安全認證應更新**：Common Criteria 等認證不應在攻擊潛力評分中獎勵設計保密
4. **推廣透明替代方案**：如 OpenTitan、Tropic Square 等開源安全硬體
5. **sec-certs 專案**（sec-certs.org）：致力於為安全認證帶來透明度

### Q&A 重點

- **是否適用於 PQC 演算法？**講者認為 ECC 的實作空間特別巨大（多層組合），PQC 或 AES 的實作多樣性可能沒那麼大，但值得研究
- **sec-certs.org 接受外部貢獻**：所有工具都在 GitHub 上開源

## 與 TWCA 臺灣網路認證的關聯

1. **認證體系的根本性反思**：TWCA 在採購安全硬體（HSM、智慧卡）時依賴 Common Criteria 和 FIPS 140 認證。本研究揭示這些認證體系系統性地獎勵保密，而保密並非有效的安全屏障。TWCA 在評估認證產品時，不應僅信任認證等級，還應要求廠商說明其實作透明度與獨立審計能力。

2. **ECC 實作的安全風險**：TWCA 的憑證簽發基礎設施大量使用 ECDSA。本研究證明安全硬體中的 ECC 實作可透過旁通道和惡意參數注入被反向工程，包括識別防禦措施類型和遮罩長度。TWCA 應確認其 HSM 中 ECC 實作是否能抵禦此類攻擊。

3. **JavaCard 平台的參數注入風險**：若 TWCA 使用基於 JavaCard 的安全元件，3Q 方法展示了透過輸入惡意曲線參數即可在不接觸旁通道的情況下識別內部防禦措施。應確認所用裝置是否對輸入參數進行完整驗證。

4. **開源安全硬體的評估**：本演講與前兩場（OpenTitan PQC 遷移、Chypnosis 攻擊）形成完整論述：封閉設計的安全硬體面臨認證體系獎勵保密、隱匿可被繞過、旁通道防護可被破解等多重問題。TWCA 在長期規劃中應認真評估 OpenTitan 等開源替代方案的可行性。
