## Abstract

介紹零知識證明（ZKP）電路的形式化驗證進展，包括自動化工具、發現的真實漏洞、已驗證的電路，以及底層的有限域 SMT 求解技術。

## Summary

### ZKP 電路為何需要驗證

ZKP 的核心是證明某個**算術電路**（有限域上的方程組）是可滿足的。應用程式必須將高層邏輯（如「合法交易」或「匿名憑證」）翻譯成電路，這個翻譯過程是 bug 的主要來源。

電路 bug 的嚴重性：

- **可偽造性**：即使只有一個方程式稍有錯誤，攻擊者就可能偽造證明（如在支付系統中無中生有地印錢）
- **不可偵測性**：電路 bug 不會破壞零知識性——ZKP 證明的東西是錯的，但仍然是零知識的。這意味著攻擊者利用漏洞時，系統的隱私保護反而使你無法察覺
- **真實案例**：Tornado Cash（基於 ZKP 的加密貨幣混幣器）曾在部署後被發現有此類 bug

研究顯示，在 141 個已知 ZKP bug 中，75% 出現在翻譯層（電路），僅 16% 在密碼學層。翻譯層的工程複雜度高、參與者多、缺乏學術規格。

### 電路驗證的核心挑戰

以 **bit decomposition**（位元分解）為例：將 32-bit 整數分解為 32 個位元的電路，看似簡單但存在微妙的 overflow 問題——當質數 P 足夠小（32-bit 以內）時，P 的位元也是合法解，導致電路不正確。

根本困難：計算是確定性的（每個輸入對應唯一輸出），但電路本質上是非確定性的（可能有多個解），這使驗證和測試都極為困難。

### 四種驗證技術

**1. Relaxation（放鬆驗證目標）**

- 完整驗證要求「等價性」：電路的每個輸入唯一確定一個輸出，且該輸出與預期函數一致
- 較弱但實用的性質：**Determinism**（確定性）——只驗證輸入唯一確定輸出，不要求與特定函數一致
- 實證數據：96% 的真實 ZKP bug 違反了 determinism，因此驗證 determinism 就能排除絕大多數 bug
- 實務優勢：驗證 determinism 不需要寫出完整的函數規格 f（審計公司客戶通常有電路但難以形式化描述電路應做什麼）

**2. SMT Solving for Finite Fields（有限域 SMT 求解）**

- 將 determinism 檢查轉化為一階邏輯公式：對電路 φ 及其副本 φ'，證明「相同輸入必然導致相同輸出」
- 問題：此前沒有 SMT 求解器能有效處理有限域
- 貢獻：將計算代數中的經典方法（Gröbner bases、minimal polynomials）適配到密碼學尺度的有限域和實際 ZKP 電路
- 限制：對極大電路可能較慢

**3. Static Analysis（靜態分析）**

- 針對特定模式的快速驗證，高度互補 SMT 求解
- 適用於已知的常見電路模式

**4. Modular Reasoning（模組化推理）**

- 將大電路分解為小片段，獨立驗證，通常可平行處理
- 這是所有高效電路驗證器的核心策略

### 產業應用

- **Veridise**（審計公司）：去年 3/4 的 ZKP 審計使用了自動形式化驗證工具，發現真實 bug 並在修復後提供無 bug 保證
- **RISC Zero**：使用這些工具驗證其 VM 電路，並整合到 CI 中持續驗證
- **Costa Group** 等其他審計團隊也在使用不同的形式化驗證工具

### 未來方向

- 互動式定理證明器（Lean、Coq）用於更手動但更深入的驗證
- Fuzzing 應用於 ZKP 電路測試
- 工具的語言無關化（支援更多 ZKP 電路語言）
- 探索更多創意性的可驗證性質，密碼學家需參與思考這些弱化性質對端到端安全性的意義
- 驗證完整堆疊（不只是電路，還包括密碼學層）

## 與 TWCA 臺灣網路認證的關聯

本場演講雖聚焦於 ZKP 電路驗證，但其方法論與洞見對 TWCA 有以下參考價值：

1. **形式化驗證方法論的借鑑**：ZKP 電路驗證所展示的 SMT 求解 + 靜態分析 + 模組化推理的組合策略，與密碼學實作驗證的思路相通。TWCA 在評估密碼學軟體（如 TLS 實作、簽章模組）的安全性時，可參考類似的分層自動驗證方法。

2. **「不可偵測的錯誤」警示**：演講強調電路 bug 因隱私保護而不可偵測——這與 TWCA 的 PKI 環境有平行之處。例如，憑證簽發邏輯中的微妙 bug 可能長期不被發現，因為正常使用情境不會觸發。形式化驗證作為預防手段的價值在此凸顯。

3. **Determinism 驗證的實用性**：不需要完整規格就能排除 96% bug 的 determinism 檢查，對 TWCA 評估第三方密碼學實作時是實用的低門檻檢查策略——在要求完整形式化規格之前，先確認實作的確定性行為。
