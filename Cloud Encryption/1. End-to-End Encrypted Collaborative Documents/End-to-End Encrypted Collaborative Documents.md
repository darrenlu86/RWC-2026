## Abstract

針對調查記者等高安全需求場景，提出一套基於安全群組訊息（如 Signal）的端對端加密協作文件系統。透過客戶端調解機制（reconciliation mechanism）與模組化安全架構，在不引入新信任假設的前提下，實現即時協作編輯，延遲僅約 120 毫秒，同時繼承底層安全訊息系統的機密性與完整性保證。

## Summary

### 研究動機與需求分析

研究團隊與國際調查記者聯盟（ICIJ）合作，針對調查記者在處理敏感資料時的協作需求進行訪談。現有工具如 Google Docs 將資料以明文儲存在伺服器上，服務提供者可存取內容，且可能被傳票要求提供或遭到入侵，這與調查資料的保密性需求不相容。

**功能需求**：
- 文件分享與存取控制
- 強收斂性（strong convergence）的協作編輯——所有使用者最終看到一致的文件狀態
- 支援非同步編輯（離線時僅透過伺服器互動即可編輯）
- 支援同步即時編輯（多人同時編輯同一位置）

**部署需求**：
- 不需要新帳號或新服務
- 不引入超出現有信任基礎的新信任假設
- 不需要自架伺服器（成本與責任問題）
- 可接受不受信任的輔助伺服器（untrusted helper server）

**安全需求**：
- 機密性與完整性（端對端加密保證）
- 存取控制（讀取、評論、寫入、管理員權限）
- 歸檔功能（將文件轉為唯讀狀態，用於法律程序）
- 安全刪除（調查結束後刪除未發布的敏感資料）
- 以上保證須在惡意伺服器下仍然成立

### 現有方案的不足

**學術方案**（Capsule、SnapDoc、基於 FHE/MPC 的方案）：有可證明安全性，但效率差（尤其 MPC 不適合即時編輯），且需要額外信任假設或新的部署環境。

**已部署方案**（CryptPad、Apple Notes 進階資料保護、Proton Docs）：功能上可行，但缺乏可證明安全性——沒有明確的安全定義，建構元件的組合方式未被分析。

### 技術架構

**核心洞見**：使用客戶端調解機制實現強收斂性——使用者只要收到相同的編輯集合（即使順序不同），最終文件狀態一致。基於此，使用者之間的互動本質上就是傳送編輯操作，因此只需保護此通道即可。

**模組化設計**：
1. **FCD（Formal Collaborative Document）**：端對端加密協作文件的形式化模型，包含 setup、新建文件、分享文件、編輯、下載等操作
2. **FAB（Formal Asynchronous Broadcast）**：抽象出通道的功能需求，證明安全的 FAB 足以建構安全的 FCD
3. **安全群組訊息**：證明安全群組訊息可以實現 FAB，安全性一路繼承到 FCD

### Signal 作為具體實例

將 FCD 操作一對一映射到 Signal 功能：
- Setup → Signal 帳號設定與連結
- 新建文件 → 建立 Signal 群組
- 分享文件 → 加入群組成員
- 編輯 → 透過 Signal 端對端加密訊息傳送編輯操作
- 下載 → 接收並本地端套用所有編輯

Signal 的安全分析（包括 Signal Protocol 和私人群組系統）直接繼承至端對端加密協作文件系統。

### 效能實驗結果

使用 Automerge 函式庫作為調解機制、Signal CLI 作為訊息介面，模擬兩位使用者以 30-120 WPM 的打字速度進行同步編輯（最壞情況：每個字元獨立加密傳送）：

- **同步編輯延遲**：端對端約 120 毫秒（95 百分位約 190 毫秒）
  - 客戶端序列化與加密：1-2 毫秒
  - 傳送至伺服器：約 50 毫秒
  - 伺服器傳播：約 100 毫秒
  - 接收端解密與整合：1-2 毫秒
- 使用者正常打字間隔為 100-200 毫秒，因此 A 的編輯在 B 的下一次按鍵前已完成傳播、解密與調解
- **大量編輯**：1 到 50,000 字元的編輯，客戶端調解時間最多 150 毫秒

### 未來方向

- 研究前向保密（forward secrecy）、後妥協安全（post-compromise security）、可否認性（deniability）在文件場景中的意義
- 分析已部署系統（特別是 CryptPad）的安全性
- 超越黑箱使用安全群組訊息，開發基於持續群組金鑰協議（continuous group key agreement）的更精細建構
- 探索使用者是否願意犧牲部分編輯功能以換取更高效的端對端加密

## 與 TWCA 臺灣網路認證的關聯

1. **安全協作文件的信任模型**：TWCA 內部處理憑證簽發、稽核報告等敏感文件時，可能使用 Google Docs 等未加密的協作工具。本研究展示了一條可行路徑——利用現有安全訊息系統（如 Signal）建構端對端加密協作文件，無需自架伺服器或引入新信任假設，值得 TWCA 評估採用於內部敏感文件處理流程。

2. **模組化安全架構的啟示**：本研究的模組化設計方法（FCD → FAB → 安全群組訊息）展示了如何透過組合已驗證的密碼學元件來建構複雜系統並繼承安全性。TWCA 在設計或評估新的安全服務時，可參考此方法論確保安全性的可追溯性。

3. **PKI 與身分驗證的整合潛力**：本系統依賴 Signal 的身分驗證機制。TWCA 作為憑證中心，未來若此類系統擴展至企業場景，可提供基於 PKI 的身分驗證與存取控制，取代 Signal 的電話號碼綁定身分模型，提升企業級安全性。
